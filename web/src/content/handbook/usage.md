# 使用说明

面向日常使用 Bedrock 的操作指引。部署、备份与升级见同页「运维手册」；开放 HTTP 接口见「开放接口」。

---

## 1. 登录与权限

1. 浏览器打开平台地址（开发默认 `http://localhost:8080` 或前端 Vite `http://localhost:8070`）。
2. 使用管理员下发的账号登录。Web 端提交加密后的 `password_cipher`，不要在页面脚本中保存明文密码。
3. 登录成功后侧栏菜单由服务端按角色下发；**看不到的菜单表示当前角色无对应权限**。
4. 进入某页面至少需要 `{菜单码}:view`（例如 `cicd_build_jobs:view`）。按钮与写操作还会校验更细的功能码。

更换角色或权限后，重新登录或刷新会话（拉取 `/auth/me`）即可看到新菜单。

## 2. 工作台

| 菜单     | 用途                                                                      |
| -------- | ------------------------------------------------------------------------- |
| 仪表盘   | 可配置卡片总览（构建、智能体运行、系统信息/状态等）；卡片本身也受权限控制 |
| 操作手册 | 本页：使用说明、运维要点与常用开放接口                                    |

---

## 3. 资源管理

先在资源管理中登记共享资源，再被 CI/CD、AI、项目等域引用。

| 菜单     | 典型操作                                                                  |
| -------- | ------------------------------------------------------------------------- |
| 代码仓库 | 登记 Git 地址、认证方式与凭证绑定                                         |
| 服务器   | 登记部署目标主机                                                          |
| 凭证     | 管理密钥/口令等；敏感字段创建后不回显，编辑留空表示不修改                 |
| 访问令牌 | 创建个人访问令牌（PAT，`br_...`），供脚本或外部系统以 Bearer 调用开放接口 |

---

## 4. CI/CD

1. **构建任务**：绑定仓库与分支，配置构建脚本、产物目录、触发方式（手动 / Webhook / Cron）及可选部署目标。
2. **手动触发**：在任务上执行「构建」，入队后异步运行。
3. **构建记录**：查看状态、日志、制品与部署尝试；进行中的运行可取消（需执行权限）。
4. **Webhook**：在任务详情查看/轮换 Webhook 密钥与 URL，按文档配置 Git 平台回调。

长任务均为异步：以状态与日志为准，勿重复狂点触发。

---

## 5. 项目管理

1. **产品项目**：创建项目、维护成员与可见范围。
2. **需求**：在项目详情中维护需求、评论与附件（侧栏可能不单独展示入口，从项目进入）。
3. **接口文档**：在项目内维护 Markdown 文档树，支持草稿、发布与导入；面向「某个业务项目」的 API 说明，与本平台开放接口文档不同。

---

## 6. AI

1. **技能（Skills）**：上传或维护可被智能体使用的技能包。
2. **智能体（Agents）**：绑定仓库/工作区、技能与运行参数；可手动或由构建事件触发运行。
3. **运行记录**：查看每次运行的状态、日志与产出。

与构建类似，智能体运行为异步任务，注意权限与工作区占用。

---

## 7. 系统管理（管理员）

| 菜单     | 用途                                  |
| -------- | ------------------------------------- |
| 用户     | 账号启停、绑定角色                    |
| 角色     | 勾选功能权限（分组 → 菜单 → 功能）    |
| 权限资源 | 查看/维护菜单与功能资源（含菜单图标） |
| 字典     | 平台字典项                            |
| 操作日志 | 审计写操作                            |

运维分组下的「进程」「开发环境」仅超级管理员可见，用于进程查看与受控脚本环境。

---

## 8. 使用注意

1. **HTTPS**：生产环境务必反向代理 HTTPS；当前会话模型在纯 HTTP 下有已知风险（见运维手册）。
2. **权限 ≠ OS 沙箱**：构建脚本、AI CLI、超管自定义命令与 Server 同 OS 用户执行。
3. **危险操作**：删除、轮换密钥、终止进程等需二次确认；写操作会记入操作日志。
