## 1. 阶段目标

跨域端到端验收、前端正式切入 embed、发布物与文档齐套，宣布 2.0 GA。

---

## 2. 范围说明

### 2.1 本阶段交付（摘要）

| 类别 | 内容 |
| --- | --- |
| 前端切换 | web-v2 切换 Gate 通过后，默认 `FRONTEND_DIR=web-v2`；CI/Release 拷贝 `web-v2/dist` → `cmd/server/dist`；旧 `web/` 保留一个发布周期作回滚 |
| 集成测试 | 登录→RBAC 菜单→构建→分发→通知→（可选）Agent→文档草稿发布；三数据库矩阵；Linux amd64/arm64 冒烟 |
| 文档 | PRD / DESIGN / ROADMAP / AGENTS 一致；操作手册与风险说明（HTTP、同 UID） |
| 发布 | 单二进制 Server + 独立 Deploy Agent；默认 SQLite 全新安装路径 |

### 2.2 明确不做（整个 2.0 GA）

- [ ] ~~1.x → 2.0 自动/离线数据迁移~~
- [ ] ~~多活水平扩展、远程构建 Runner、S3 对象存储~~
- [ ] ~~容量/延迟 SLO 验收~~
- [ ] ~~显式 deny 权限、非项目域对象 ACL~~

### 2.3 与前期阶段关系

- 本阶段**不新开业务域**；以集成、切换、发布与文档收敛为主。
- 发现 P0–P4 Gate 回退项时，先修复对应阶段缺口，再关闭 P5。

## 3. Gate 条件

### 3.1 web-v2 切换 Gate（必须全部满足）

来源：[ROADMAP.md §7.3](../ROADMAP.md#73-web-v2-切换-gate必须全部满足)

1. 路由深链与书签路径可用（含构建详情等）。
2. API 信封、401 refresh、登出跳转与旧行为一致。
3. `password_cipher` 字节级兼容；`__BEDROCK_ENCRYPTION_KEY__` 注入有效。
4. WS：构建日志与通知；Agent 日志（若已启用）。
5. 制品下载、备份/恢复或等价 FormData 上传下载可用。
6. 菜单完全由服务端下发，无本地硬编码全量菜单。
7. `bun run lint && bun run build`（或 `vp check && vp build`）零错误；`go build` embed 成功。

### 3.2 退出 Gate（2.0 GA）

来源：[ROADMAP.md §7.5](../ROADMAP.md#75-退出-gate20-ga)

1. P0–P4 各 Gate 全部通过且无未关闭的落地阻塞项。
2. web-v2 为默认嵌入前端；回滚说明已文档化。
3. 模块验收清单（[PRD §20](../PRD.md#20-模块验收清单汇总) 语义，按 DESIGN 固化后的行为）全部勾选。
4. 已知风险（HTTP、同 UID、自定义超管命令）在产品内可见且写入 AGENTS/DESIGN。

另须满足 [ROADMAP.md §9 通用质量门禁](../ROADMAP.md#9-每阶段通用质量门禁)。

## 4. 详细开发任务清单

### 4.1 前端切换与构建链

- [ ] 确认 web-v2 切换 Gate（§3.1）七项全部通过并留存证据（测试记录/CI）
- [ ] Makefile / CI：默认 `FRONTEND_DIR=web-v2`
- [ ] Release：构建 `web-v2/dist` → 拷贝至 `cmd/server/dist` → `go build` embed
- [ ] 文档化旧 `web/`（或上一版产物）保留一个发布周期的回滚步骤
- [ ] 验证 Linux amd64 / arm64 交叉编译目标与发布包命名
- [ ] 验证独立 Deploy Agent 二进制同版本发布物

### 4.2 端到端集成测试

- [ ] E2E 主路径：登录 → 菜单按角色下发 → 创建仓库/任务 → 构建 → 归档下载 → 分发（含失败 summary）→ 通知
- [ ] E2E（可选但 GA 建议）：构建事件 → AgentRun → 日志 WS
- [ ] E2E：项目文档生成 → draft → diff → publish（含 409 场景）
- [ ] E2E：PAT 下载 Skill / 触发 agents:run
- [ ] 三数据库矩阵冒烟：sqlite / postgres / mysql 全新安装启动 + 关键路径
- [ ] 重启恢复回归：BuildRun / AgentRun / 安装任务 queued 与 interrupted
- [ ] Playwright（或等价）冒烟：登录、菜单、构建日志（对齐 DESIGN §13）
- [ ] Linux amd64/arm64 发布包冒烟（至少启动 + 健康/登录）

### 4.3 契约与回归清单收敛

- [ ] 全量 OpenAPI 与实现合同测试无阻塞失败
- [ ] 投影生成纳入 CI；禁止手改投影的策略有文档/检查
- [ ] 对照 [P1](./P1.md)–[P4](./P4.md) 验收清单，关闭遗留项或明确非阻塞已知问题列表
- [ ] 按 [PRD §20](../PRD.md#20-模块验收清单汇总) 做模块勾选，行为冲突处以 DESIGN 为准并回填文档

### 4.4 文档与产品内风险说明

- [ ] 对齐 PRD / DESIGN / ROADMAP / AGENTS：分期、Gate、禁止事项无矛盾
- [ ] 操作手册：全新安装（默认 SQLite）、多库配置、备份指引（不假装统一物理备份）
- [ ] 产品内可见风险：HTTP + localStorage Bearer；构建/AI 同 UID；自定义超管命令
- [ ] 显著声明：**不提供** 1.x → 2.0 数据迁移
- [ ] 回滚说明：前端 embed 回滚；发布包回退路径
- [ ] 更新 AGENTS 常用命令与目录结构至 GA 实态

### 4.5 发布准备

- [ ] 默认全新安装路径验证（空 data 目录 → migration → 超管种子 → 登录）
- [ ] 发布检查单：版本号、changelog、校验和（按仓库惯例）
- [ ] GA 宣布前确认无未关闭落地阻塞项（P0–P4 Gate + 切换 Gate）

### 4.6 明确不做项的护栏（防回退）

- [ ] CI/文档/代码审查检查：无 1.x 静默迁移脚本作为正式路径
- [ ] 确认未引入多活粘滞、远程 Runner、S3 作为 GA 必达
- [ ] 确认无容量/延迟 SLO 验收阻塞 GA
- [ ] 确认无显式 deny 权限模型、无非项目域对象 ACL

## 5. 验收清单

### 5.1 web-v2 切换 Gate

- [ ] 路由深链/书签可用（含构建详情等关键页）
- [ ] API 信封、401 refresh、登出跳转行为正确
- [ ] `password_cipher` 字节级兼容；embed 注入 `__BEDROCK_ENCRYPTION_KEY__` 有效
- [ ] WS：构建日志与通知可用；Agent 日志在启用时可用
- [ ] 制品下载可用；备份/恢复或等价 FormData 上传下载可用
- [ ] 菜单完全由服务端下发，无本地硬编码全量菜单
- [ ] `vp check && vp build`（或 `bun run lint && bun run build`）零错误
- [ ] `go build`（含 embed）成功

### 5.2 2.0 GA 退出 Gate

- [ ] [P0](../ROADMAP.md#24-退出-gate) / [P1](./P1.md#3-gate-条件退出准则) / [P2](./P2.md#3-gate-条件退出准则) / [P3](./P3.md#3-gate-条件退出准则) / [P4](./P4.md#3-gate-条件退出准则) 全部通过
- [ ] 无未关闭落地阻塞项（或已文档化为明确非阻塞已知问题且产品接受）
- [ ] 默认嵌入前端为 web-v2；回滚说明已文档化
- [ ] PRD §20 模块验收清单（按 DESIGN 行为）全部勾选：
  - [ ] 部署/DB：SQLite 默认；切 Postgres/MySQL 重启可用；错误配置拒绝启动
  - [ ] 权限：`{path}:action`；菜单下发；图标 ≤32KB；运维仅超管；项目 ACL + 全局 RBAC
  - [ ] 仪表盘：排序显隐；权限过滤
  - [ ] 运维：进程；工具链；源回退
  - [ ] CI/CD：一仓多任务；分发失败不改构建成功；凭证 RBAC
  - [ ] 项目：成员角色；需求；文档树；生成→draft→发布
  - [ ] AI：独立 AgentRun；四类触发；上下文=提示词+仓库
  - [ ] Skills：ZIP；公私；覆盖更新；PAT 下载
  - [ ] 系统：用户角色资源字典操作日志
- [ ] 已知风险在产品内可见且已写入 AGENTS/DESIGN

### 5.3 集成 / 发布补充验收

- [ ] 端到端主路径在至少一种环境跑通并留存记录
- [ ] 三数据库矩阵冒烟通过
- [ ] Linux amd64 与 arm64 发布包冒烟通过
- [ ] Server 单二进制 + Deploy Agent 独立二进制齐套
- [ ] 默认 SQLite 全新安装路径可复现

### 5.4 安全边界与禁止事项

- [ ] 文档未声称 HTTP/localStorage 具备防窃听传输安全
- [ ] 文档未声称同 UID 构建/AI 为沙箱隔离
- [ ] 未提供 1.x 数据迁移作为支持路径
- [ ] 未将分发失败写为 BuildRun `failed`
- [ ] 未在流水线内同步执行 Agent
