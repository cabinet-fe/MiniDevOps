## 1. 阶段目标

用可配置 RBAC 替换固定三角色；将 1.x `Project/Environment/Build` 演进为 `Repository / BuildJob / BuildRun`；完成构建—归档—分发主路径。

---

## 2. 范围说明

### 2.1 本阶段交付（摘要）

| 类别 | 内容 |
| --- | --- |
| 权限 | Role、RolePermission、RbacResource、MenuMetadata；多角色并集；`RequirePermission`；登录/me 下发裁剪菜单树；父级由可见后代自动补齐；一级菜单图标 Base64 ≤32KB；运维 path 仅超管 |
| CI/CD | Repository、BuildJob、DeployTarget（Job 私有 1:N）、BuildRun、BuildDeployAttempt；Pipeline 适配；Scheduler/Cron（IANA 时区、禁重叠、停机跳过）；Webhook（签名优先 + URL secret fallback + delivery 去重） |
| 状态机 | `status` 与 `stage` 分离；归档成功 → `status=success`；分发更新 `distribution_summary`；redeploy 追加 attempt；Run 最小配置快照；queued 恢复 / running→interrupted |
| 凭证 | Credential 通用库 + `credential:use` **绑定/修改时**校验；执行侧仅需任务 `execute` |
| 前端 | 菜单驱动侧栏；仓库/任务/执行/服务器/凭证页面；构建日志 WS；重新分发 UI |
| 测试 | 权限矩阵；Webhook 多平台；分发失败不改构建成功；重启恢复；三库 CRUD 合同测试 |

## 3. Gate 条件（退出准则）

来源：[ROADMAP.md §3.4](../ROADMAP.md#34-退出-gate)

1. 自定义角色勾选 `{path}:action` 后，菜单/路由/API 一致生效；运维对非超管恒 403。
2. 同一仓库 ≥2 个 BuildJob 可分别执行；制品可下载。
3. 部署失败时 BuildRun 仍为 `success`，summary 为 `partial` / `all_failed`；redeploy 追加 `BuildDeployAttempt` 且 summary 更新为最新结果。
4. Webhook：平台签名校验优先；generic/手动仍可用 URL secret；日志脱敏。
5. Cron：每任务 IANA 时区；同任务禁止重叠；停机错过的触发跳过。
6. 服务重启：queued 重新入队；running → interrupted，可人工 retry。

另须满足 [ROADMAP.md §9 通用质量门禁](../ROADMAP.md#9-每阶段通用质量门禁)。

## 4. 详细开发任务清单

### 4.1 契约与数据模型

- [ ] 在 OpenAPI 3.2 源补充/对齐：Users、Roles、RBAC resources、Menus、Dictionaries、Operation logs
- [ ] 在 OpenAPI 补充/对齐：Repositories、BuildJobs、DeployTargets、BuildRuns、BuildDeployAttempts、Servers、Credentials、Webhook
- [ ] 再生 `api/openapi.3.1.projection.yaml`（禁止手改投影）
- [ ] 版本化 migration：`Role` / `RolePermission` / `RbacResource` / `MenuMetadata`
- [ ] 版本化 migration：`Repository` / `BuildJob` / `DeployTarget` / `BuildRun` / `BuildDeployAttempt` / `Server` / `Credential`
- [ ] 种子：内置超管；预置菜单资源树（含 `cicd.*`、`system.*`；运维 path 可预置但仅超管生效）
- [ ] 移除/停用固定三角色（admin/ops/dev）作为权限真源；迁移到可配置 Role（无 1.x 数据迁移，全新安装种子即可）

### 4.2 后端 — RBAC / 系统管理（`internal/rbac`、`internal/system`、`internal/auth`）

- [ ] Role CRUD；RolePermission 绑定 `{path}:action` 集合
- [ ] RbacResource 树 CRUD（menu/page/action/card）；与 MenuMetadata 一对一
- [ ] 权限并集计算（用户多角色）；`RequirePermission` 中间件
- [ ] 运维 path 硬门控：非超管即使误勾选仍 403
- [ ] 登录 / `GET /auth/me` 返回：用户信息、权限码、**裁剪菜单树**（含一级图标 Base64）
- [ ] 父级菜单：存在可见后代则自动补齐分组；叶子需自身 `:view`
- [ ] 一级菜单图标：上传/更新 Base64；原始体积 >32KB → 400
- [ ] 用户 CRUD（禁用后后续请求失败）；字典 CRUD；操作日志查询（写操作审计）
- [ ] 菜单维护 API（含 `PUT .../menus/:id/icon`）

### 4.3 后端 — CI/CD 领域（`internal/cicd`）

- [ ] Repository CRUD；测试拉取/列分支；Webhook secret 展示与轮换；被 Job 引用后删除保护
- [ ] BuildJob CRUD（属仓库 1:N）；分支策略、脚本、工作目录、输出、环境变量名、触发、制品保留
- [ ] DeployTarget：归属 BuildJob 私有 1:N（顺序、服务器、路径、方法、部署后脚本）
- [ ] Server CRUD；连接测试；被 DeployTarget 引用保护
- [ ] Credential 通用库（password/token/ssh_key/api_key）；AES-GCM；永不回显明文
- [ ] 绑定/修改凭证引用时校验 `cicd.credentials:use`；执行侧仅校验任务 `execute`
- [ ] Credential 删除保护（仍被引用时 409 + 提示）

### 4.4 后端 — 引擎与状态机（`internal/engine`、`internal/deployer`）

- [ ] Pipeline 适配新模型：克隆 → 构建 → 归档 →（标记 success，可下载）→ 按 DeployTarget 顺序分发
- [ ] `status` 与 `stage` 分字段（见 [DESIGN §5.2](../DESIGN.md#52-cicd-状态机)）
- [ ] 归档成功：`status=success`，制品可下载；**禁止**因分发失败改 `failed`
- [ ] 更新 `distribution_summary`：`none` | `running` | `all_success` | `partial` | `all_failed` | `cancelled`
- [ ] `BuildDeployAttempt`：每次分发/重新分发追加；含目标快照、状态、日志引用
- [ ] `redeploy`：同一 BuildRun，追加 attempt，summary 反映最新一批
- [ ] `retry`：新建 BuildRun
- [ ] Run 创建写入最小配置快照（commit、脚本 SHA、变量名、DeployTarget 副本等）
- [ ] 取消：构建中 → `cancelled`；已 success 仅取消分发 → 保持 success + summary
- [ ] **禁止**流水线内同步执行 Agent（事件钩子可留接口，实现属 P4）
- [ ] 五法部署器复用：`rsync` / `sftp` / `scp` / `agent` / `local`
- [ ] Scheduler：全局并发上限；DB 持久化 queued；重启 queued 恢复、running → `interrupted`
- [ ] Cron：每任务 IANA `timezone`；同任务非终态禁重叠（跳过并审计）；停机错过不补跑
- [ ] Webhook：`POST /api/v1/webhook/repos/:repository_id/:secret`；签名优先；URL secret fallback；delivery 去重；日志脱敏
- [ ] 构建日志文件 + WS 频道；终态站内通知（可复用现有通道）

### 4.5 前端（`web-v2/`）

- [ ] 侧栏仅渲染 `/auth/me` 下发菜单树（禁止硬编码全量菜单再隐藏）
- [ ] 路由守卫 + 按钮级 `hasPermission`（composables）
- [ ] 系统管理：用户 / 角色权限勾选 / 资源树 / 菜单（含一级图标上传）/ 字典 / 操作日志
- [ ] CI/CD：仓库、构建任务（含 DeployTarget）、构建执行列表/详情、服务器、凭证
- [ ] 构建日志 WebSocket 实时流
- [ ] 制品下载；取消 / 重试 / **重新分发** UI
- [ ] 敏感字段编辑留空表示不改；密文不回显
- [ ] HTTP 统一走 `@cat-kit/http`；UI 优先 `@veltra/*`

### 4.6 测试

- [ ] 权限矩阵单测/集成：并集、运维硬门控、菜单裁剪与父级补齐
- [ ] 图标 >32KB 拒绝；无 `:view` 菜单不下发且 API 403
- [ ] Webhook：多平台签名校验、generic 仅 secret、重复 delivery 幂等、日志无 secret
- [ ] 状态机：分发失败仍 `success` + summary；redeploy 追加 attempt
- [ ] 重启恢复：queued 重入队；running → interrupted；可人工 retry
- [ ] Cron：时区、重叠跳过（可用可控时钟/注入）
- [ ] 三库 CRUD 合同测试（Repository/Job/Run/Credential 等关键表）
- [ ] OpenAPI 合同测试脚手架覆盖本阶段新增路径

### 4.7 文档与说明

- [ ] 用户可见说明：同 UID 脚本执行风险；限制脚本编辑权限并审计

## 5. 验收清单

### 5.1 Gate 验收

- [ ] 自定义角色勾选权限后，菜单、路由、API 行为一致
- [ ] 非超管访问运维相关 path / API 恒 403（含角色误勾选场景）
- [ ] 同一仓库配置 ≥2 个 BuildJob，可分别触发执行
- [ ] 归档成功后制品可下载；`status=success`
- [ ] 部署失败时 BuildRun 仍为 `success`，`distribution_summary` 为 `partial` 或 `all_failed`
- [ ] 重新分发追加 `BuildDeployAttempt`，summary 更新为最新结果（不新建 BuildRun）
- [ ] Webhook：有平台签名时校验失败拒绝；generic/手动可用 URL secret；日志脱敏
- [ ] Cron：IANA 时区生效；同任务重叠触发被跳过；停机错过的触发不补跑
- [ ] 服务重启：queued 重新入队；running → interrupted；可人工 retry 成功

### 5.2 功能补充验收（对齐 PRD §9.7 / DESIGN）

- [ ] Webhook 按分支规则匹配对应 BuildJob
- [ ] 无 `credentials:use` 的用户无法在绑定场景盗用凭证；API 不回显密文
- [ ] 五法部署至少覆盖可测路径（含 local 与一种远程方法）
- [ ] 前端侧栏无硬编码全量菜单；换角色后 `/auth/me` 刷新即生效
- [ ] 一级菜单图标 ≤32KB；超限明确错误

### 5.3 契约 / 测试 / 质量门禁

- [ ] OpenAPI 3.2 已更新且投影已再生
- [ ] 表结构变更有版本化 migration；三驱动合同测试通过
- [ ] 写操作有审计；敏感字段不落日志
- [ ] 未引入「部署失败改构建失败」「流水线内同步 Agent」「1.x 自动迁移」

### 5.4 安全边界表述

- [ ] 文档/产品说明未声称构建沙箱隔离
- [ ] 同 UID 执行风险对用户可见
