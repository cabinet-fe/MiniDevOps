## 1. 阶段目标

独立产品协作域：产品项目、成员角色、需求列表、Markdown 接口文档树；与 CI/CD 松耦合可选关联。

---

## 2. 范围说明

### 2.1 本阶段交付（摘要）

| 类别 | 内容 |
| --- | --- |
| 项目 | ProductProject、ProjectMember（Owner/Admin/Member/Readonly）；显式 `project_projects:view_all` / `manage_all` |
| 需求 | Requirement、评论、附件（走 StorageObject）；状态字典可扩展 |
| 文档 | ApiDocNode 树；同节点 `published_content` + `draft_content` + 版本字段；上传/移动/删除；发布 `expected_version` 乐观锁 |
| 权限 | 全局 RBAC **且** 项目成员 ACL（无 view_all/manage_all 时）；manage_all 可管理全部项目且无需加入 |
| 前端 | 项目列表/详情、成员、需求、文档树、草稿 diff 与发布确认 |
| 测试 | 非成员不可见；view_all/manage_all 行为；并发发布 409；附件配额与 XSS 防护 |

### 2.3 与 P4 的接口（本阶段必须完成草稿模型）

来源：[ROADMAP.md §5.4](../ROADMAP.md#54-与-p4-的接口)

- [x] `POST .../docs/generate` **可定义契约**；实现可返回「依赖 AI 域」或 stub，P4 接通后启用
- [x] 草稿 / 发布模型（published + draft + `expected_version`）**本阶段必须完成**，不依赖 Agent
- [x] 并行说明：后端可与 P4 部分并行；**P5/GA 要求 P3 与 P4 Gate 均通过**

### 2.4 ACL 鉴权公式（实现真源）

见 [DESIGN.md §4.4](../DESIGN.md#44-项目-acl唯一对象级-acl)：

```text
允许 = 全局功能权限(path:action)
     AND (
           超管
        OR 持有 project_projects:view_all / manage_all（按动作）
        OR 是项目成员且项目角色允许该动作
         )
```

- 无 `view_all`：列表仅返回自己加入的项目
- `manage_all`：可管理全部项目成员与内容，**无需**加入项目
- 普通 `:update` **不**隐含全局越权

## 3. Gate 条件（退出准则）

来源：[ROADMAP.md §5.5](../ROADMAP.md#55-退出-gate)

1. 成员角色能力符合 PRD；普通用户仅见加入的项目。
2. `view_all` 可见全部；`manage_all` 可管理成员与内容且无需加入项目；普通 `:update` **不**隐含全局越权。
3. 多级文档树、上传 `.md`/压缩包导入可用。
4. 生成结果只写 draft；发布需确认；`expected_version` 冲突返回 409；可查看 diff 摘要。

另须满足 [ROADMAP.md §9 通用质量门禁](../ROADMAP.md#9-每阶段通用质量门禁)。

## 4. 详细开发任务清单

### 4.1 契约与数据模型

- [x] OpenAPI：Projects、Members、Requirements、Comments、Attachments、Docs nodes、upload、publish、diff；`docs/generate` 契约（可标依赖 AI）
- [x] 再生 3.1 投影
- [x] migration：`ProductProject`、`ProjectMember`
- [x] migration：`Requirement`、`RequirementComment`、附件引用（StorageObject）
- [x] migration：`ApiDocNode`（树 + published/draft/版本字段，见 DESIGN §5.4）
- [x] 种子：`project_*` 菜单与权限资源，含 `project_projects:view_all`、`manage_all`
- [x] 字典：需求状态基线 `backlog` / `todo` / `doing` / `done` / `cancelled`（可扩展）

### 4.2 后端 — 项目与成员（`internal/project`）

- [x] ProductProject CRUD：名称、slug 唯一、描述、状态（活跃/归档）、负责人、标签
- [x] 创建者默认为 Owner
- [x] ProjectMember CRUD：Owner / Admin / Member / Readonly
- [x] Owner 转让：仅当前 Owner 或 `manage_all`
- [x] 列表过滤：无 `view_all` 仅加入的项目；`view_all` 可见全部
- [x] `manage_all`：可管理成员与内容且无需加入；全局 `:update` 不绕过成员范围
- [x] 归档/解散：受全局 RBAC + 项目角色约束
- [x] 可选关联：需求/文档生成可选引用 Repository（无强制外键）

### 4.3 后端 — 需求

- [x] Requirement CRUD：标题、描述、状态、优先级、负责人、标签
- [x] 列表筛选、分页、排序；状态流转（固定集合 + 字典扩展校验）
- [x] 评论 CRUD
- [x] 附件上传/下载：经 StorageService / StorageObject；默认限额 20MB
- [x] 权限：全局 `project.requirements:*` **且** 项目角色允许

### 4.4 后端 — 文档树与发布

- [x] ApiDocNode 树：目录 + 文档节点；创建/重命名/移动排序/删除
- [x] 双态字段：`published_content`、`draft_content`、`content_version`、`draft_base_version`、`draft_updated_at`、可选 `draft_source_run_id`
- [x] 上传 `.md`；压缩包导入（防 Zip Slip / 解压炸弹；导入限额 100MB）
- [x] 编辑/导入更新：**只写 draft**
- [x] `POST .../publish`：body `expected_version`；匹配则 draft→published、`content_version++`、清空 draft；冲突 **409**
- [x] diff 摘要 API：published vs draft
- [x] `POST .../docs/generate`：契约落地；本阶段 stub 或明确「依赖 AI 域」错误/占位（P4 接通）
- [x] Markdown 渲染侧 XSS 消毒（前端强制；API 可标记）

### 4.5 前端（`web/`）

- [x] 项目列表/详情；归档态展示
- [x] 成员管理（角色变更、转让 Owner、按 ACL 隐藏无权限操作）
- [x] 需求列表/详情：筛选、评论、附件
- [x] 文档树：多级导航、上传 `.md`/压缩包、移动删除
- [x] 草稿编辑；diff 摘要展示；发布确认对话框（含版本冲突处理）
- [x] 生成入口可先禁用或提示依赖 AI（与契约一致）
- [x] 菜单由服务端下发 `project_*`；无硬编码全量菜单

### 4.6 测试

- [x] 非成员且无 view_all：项目不可见、详情 403/404 策略一致
- [x] `view_all`：可见全部项目只读范围符合设计；`manage_all`：无需加入即可管成员与内容
- [x] 普通 `project_projects:update` **不能**管理未加入项目
- [x] Owner/Admin/Member/Readonly 能力矩阵（对齐 PRD §8.1）
- [x] 并发发布：错误 `expected_version` → 409
- [x] 附件超限拒绝；ZIP 路径穿越拒绝
- [x] 文档导入只改 draft，不直接改 published
- [x] 三库合同测试（项目/成员/文档节点）

### 4.7 文档

- [x] 说明项目域为唯一对象级 ACL；CI/CD 等仍为全局 RBAC only
- [x] 文档生成与 AI 的衔接点写入设计/操作说明（指向 P4）

## 5. 验收清单

### 5.1 Gate 验收

- [x] 项目成员四角色能力符合 PRD（Owner/Admin/Member/Readonly）
- [x] 普通用户（无 view_all）仅见自己加入的项目
- [x] `view_all` 可见全部项目
- [x] `manage_all` 可管理成员与内容且无需加入项目
- [x] 普通 `:update` 不隐含全局越权（未加入项目不可管）
- [x] 可构建多级文档树；上传 `.md` 与压缩包导入可用
- [x] 内容变更只写 draft；发布需确认
- [x] `expected_version` 冲突返回 409
- [x] 可查看 published vs draft 的 diff 摘要

### 5.2 功能补充验收

- [x] 需求 CRUD、评论、附件可用；状态可由字典扩展但不需任意状态机配置器
- [x] 与 CI/CD 仓库关联为可选，非强制
- [x] `docs/generate` 契约存在；未接通 AI 时行为明确（stub/错误），不破坏草稿模型
- [x] 附件与导入走统一存储，遵守默认限额

### 5.3 契约 / 测试 / 质量门禁

- [x] OpenAPI 已更新且投影已再生
- [x] 版本化 migration + 三驱动合同测试
- [x] 写操作有审计；敏感字段不落日志
- [x] 未引入 Skill 安装器、强制仓库绑定、任意状态机配置器
- [x] 未为非项目域引入对象 ACL

### 5.4 安全边界表述

- [x] Markdown XSS 在渲染侧有消毒策略
- [x] Zip 导入有防 Zip Slip / 解压炸弹控制
