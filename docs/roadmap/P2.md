## 1. 阶段目标

可配置仪表盘三卡片；超管运维：进程管理 + 开发工具链（含多安装源）。

---

## 2. 范围说明

### 2.1 本阶段交付（摘要）

| 类别 | 内容 |
| --- | --- |
| 仪表盘 | DashboardLayout；构建摘要 / 系统信息 / 系统状态卡片；按资源权限过滤；用户排序与显隐持久化 |
| 系统信息 | 非超管可见**完整只读**系统信息（与超管信息字段一致），但无运维写操作 |
| 运维 | 进程列表/终止（保护自身与危险进程）；ToolchainDefinition；InstallSource 优先级回退；ToolchainInstallJob（同重启协议） |
| 自定义工具链 | **仅超管**可创建/修改/执行；命令快照 + 强提示 + 完整审计 |
| 前端 | 仪表盘编辑；运维进程页；工具链与安装源页；安装任务日志 |
| 测试 | 无权限卡片不可见；非超管运维 API 403；源回退可观测 |

### 2.2 明确不做

- [ ] ~~自定义仪表盘任意组件市场~~
- [ ] ~~全文日志检索、操作日志导出~~（查询即可）
- [ ] ~~AI / 项目管理~~ → **P3 / P4**

### 2.3 与 P1 的边界

- 依赖 P1 的 RBAC（卡片权限码、运维硬门控）与 CI/CD 构建摘要数据源。
- 若 P1 仅留了仪表盘只读 stub，本阶段替换为完整可配置布局。

---

## 3. Gate 条件（退出准则）

来源：[ROADMAP.md §4.4](../ROADMAP.md#44-退出-gate)

1. 无 `cicd.build_runs:view` 的角色看不到构建卡片且无法添加。
2. 布局刷新后保持；状态卡片按合理频率刷新且不阻塞首屏。
3. 超管可检测/安装/升级/卸载/切版本；第一源失败第二源成功时可追踪。
4. 非超管调用运维 API 恒 403；自定义命令非超管不可维护。

另须满足 [ROADMAP.md §9 通用质量门禁](../ROADMAP.md#9-每阶段通用质量门禁)。

## 4. 详细开发任务清单

### 4.1 契约与数据模型

- [x] OpenAPI：Dashboard layout + build-summary / system-info / system-status
- [x] OpenAPI：Ops processes、toolchains、sources、install jobs
- [x] 再生 3.1 投影
- [x] migration：`DashboardLayout`（按用户持久化卡片排序/显隐）
- [x] migration：`ToolchainDefinition`、`InstallSource`、`ToolchainInstallJob`
- [x] 种子：预置仪表盘卡片资源（`dashboard`、`dashboard.system_info`、`dashboard.system_status` 等）与运维菜单资源
- [x] 种子：内置工具链定义（Go / Node.js / Java / Python）及默认安装源优先级

### 4.2 后端 — 仪表盘（`internal/dashboard`）

- [x] `GET/PUT /dashboard/layout`：用户级排序与显隐；无权限卡片加载时过滤且不可添加
- [x] `GET /dashboard/build-summary`：运行中/排队、最近结果、成功率摘要（需 `cicd.build_runs:view`）
- [x] `GET /dashboard/system-info`：版本、OS、架构、运行时、启动时间、主机名等；**非超管完整只读**（与超管字段一致）
- [x] `GET /dashboard/system-status`：CPU/内存/磁盘、服务健康、关键目录空间（需对应 `:view`）
- [x] 进入仪表盘需 `dashboard:view`；各卡片另校验绑定 path 的 `:view`
- [x] 默认布局：对有权限用户预置三类卡片

### 4.3 后端 — 运维（`internal/ops`）

- [x] 运维 API 统一超管门控（与 RBAC 运维 path 硬拒绝一致）
- [x] 进程列表：PID、名称、CPU、内存、用户、启动时间、监听端口；筛选与排序
- [x] 终止进程：禁止终止 Bedrock 自身关键进程；危险系统进程黑名单
- [x] ToolchainDefinition：内置 + 自定义；检测命令、安装/升级/卸载模板、版本列表来源
- [x] InstallSource：多源优先级；连通性 ping；安装按优先级尝试并记录失败原因
- [x] 操作：detect / install / upgrade / uninstall / switch 默认版本
- [x] ToolchainInstallJob：异步任务；queued 持久化；重启协议对齐 DESIGN（running→interrupted/failed，保留日志；重试新任务）
- [x] 自定义工具链：**仅超管** CRUD 与执行；命令快照；强提示；完整审计（OperationLog）
- [x] 安装任务日志可查询（HTTP 轮询）

### 4.4 前端（`web-v2/`）

- [x] 仪表盘页：三卡片渲染；编辑模式（显隐、排序）；刷新后保持
- [x] 构建摘要卡片：跳转构建执行详情
- [x] 系统信息 / 系统状态卡片；状态卡轮询节流，不阻塞首屏
- [x] 无权限卡片不展示、编辑器中不可添加
- [x] 运维：进程管理页（列表/筛选/终止 + 二次确认）
- [x] 运维：工具链列表、检测、安装/升级/卸载/切版本
- [x] 安装源管理与连通性检测 UI
- [x] 自定义工具链表单：强风险提示；仅超管可见入口
- [x] 安装任务状态与日志查看

### 4.5 测试

- [x] 无 `cicd.build_runs:view` → 构建卡片不可见且无法添加（含历史布局残留过滤）
- [x] 布局 PUT 后重启/重登仍保持
- [x] 系统状态刷新节流行为可测（不阻塞首屏接口）
- [x] 非超管运维 API 恒 403；自定义工具链写接口非超管 403
- [x] 终止自身进程被拒绝；危险进程黑名单生效
- [x] 第一源失败、第二源成功：任务 success 且日志可见回退
- [x] 安装任务重启：interrupted/日志保留；可人工重试新任务
- [x] 三库 migration / 关键 CRUD 合同测试

### 4.6 文档

- [x] 自定义超管命令风险写入用户可见说明（与 AGENTS/DESIGN 已接受风险一致）
- [x] 明确：系统信息对非超管为完整只读，不等于开放运维写操作

## 5. 验收清单

### 5.1 Gate 验收

- [x] 无 `cicd.build_runs:view` 的角色看不到构建任务卡片且无法添加
- [x] 用户调整卡片顺序/显隐后刷新（或重新登录）仍保持
- [x] 系统状态卡片按合理频率刷新，不阻塞仪表盘首屏
- [x] 超管可完成检测、安装、升级、卸载、切版本（至少一种内置工具链可测通）
- [x] 第一源失败、第二源成功时任务成功且日志可追踪回退
- [x] 非超管调用运维 API 恒 403
- [x] 自定义工具链命令非超管不可创建/修改/执行

### 5.2 功能补充验收（对齐 PRD §6.4、§7）

- [x] 非超管可见完整只读系统信息字段，但无进程终止/工具链写操作入口
- [x] 超管可查询进程并终止普通用户进程（OS 权限允许范围内）
- [x] 尝试终止 Bedrock 自身服务进程被拒绝
- [x] 安装源连通性检测可用
- [x] 自定义工具定义可被检测与安装流程复用

### 5.3 契约 / 测试 / 质量门禁

- [x] OpenAPI 已更新且投影已再生
- [x] 版本化 migration + 三驱动合同测试
- [x] 写操作（含自定义命令、安装、终止进程）有审计；敏感信息不落日志
- [x] 未引入组件市场、全文日志检索导出、AI/项目域功能

### 5.4 安全边界表述

- [x] 自定义命令风险提示在 UI/文档可见
- [x] 未将同 UID 安装/执行描述为沙箱隔离
