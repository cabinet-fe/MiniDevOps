# 重设计项目详情页信息密度与链接展示

> 状态: 已执行

## 目标

重构 `web/src/pages/projects/detail.tsx` 的项目详情区块，减少无效留白与说明性赘述，用更紧凑但不拥挤的结构展示核心信息；同时优化仓库地址、Webhook URL 等长链接的视觉与交互，尽量保持单行展示，并提供清晰的复制与打开能力。

## 内容

### 步骤 1：压缩项目总览与基础信息布局

- 检查当前 hero、仓库信息卡片、Webhook 卡片的层级与留白，改为更紧凑的横向信息编排，减少仅用于装饰的说明文字和大块空区域。
- 保留现有项目标签、分组、构建统计和常用操作，但提升首屏可读性，让项目名称、关键属性、核心入口更集中。

### 步骤 2：重构长链接与 webhook 信息展示

- 针对仓库地址、Webhook URL、JSONPath 等长文本字段设计单行优先的展示组件，使用截断、省略、复制按钮、外链按钮或悬浮提示等方式避免频繁换行。
- 调整 Webhook 指引区域，保留必要配置提示，但减少冗余描述，让入口、类型、请求头和路径配置更优雅地集中展示。

### 步骤 3：验证页面实现与构建质量

- 运行前端 lint 与构建，确认 `web/src/pages/projects/detail.tsx` 的改动通过检查，且未引入明显的类型或构建错误。

## 影响范围

- `web/src/pages/projects/detail.tsx`
