# 优化 Go 嵌入静态资源缓存策略

> 状态: 已执行

## 目标

为 Go `embed` 托管的前端静态资源补充合理的 HTTP 缓存策略，减少重复下载并保证入口页面可及时更新。

## 内容

1. 梳理生产环境 `serveSPA` 的静态资源返回路径，明确 `index.html` 与普通静态文件的响应分支。
2. 在生产环境静态文件响应中添加缓存控制：`index.html` 使用禁止强缓存策略，带构建指纹的静态资源使用长期缓存策略，其它静态资源使用短期缓存策略。
3. 运行相关 Go 测试/编译检查，确认改动不会引入构建错误。

## 影响范围

- `cmd/server/embed_prod.go`：为嵌入静态资源新增缓存策略（`index.html` 禁止强缓存、指纹资源长期缓存、其它静态资源短期缓存）。
