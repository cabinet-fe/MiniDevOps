# 构建详情页完善与核心构建体验优化

> 状态: 未执行

## 目标

完善当前极简的构建详情页（`builds/detail.tsx` 仅 27 行占位），打造完整的构建详情体验，包括构建元信息展示、实时日志查看器增强（ANSI 颜色、搜索过滤）、产物下载、部署/回滚/取消操作，以及构建状态时间线。同时修复路由与页面参数不匹配的 bug。

## 内容

### 步骤 1：修复构建详情页路由参数

当前路由 `builds/:id` 只有一个 `:id` 参数，但 `detail.tsx` 中 `useParams()` 解构了 `id` 和 `buildId` 两个参数，导致逻辑错误。

- 修改 `App.tsx` 路由，调整构建详情的路由使其参数正确。注意当前项目详情页的构建列表中链接为 `/projects/${projectId}/builds/${b.id}`，需保持一致。
- 更新 `detail.tsx` 中 `useParams` 的使用。

**完成标准**：从项目详情页点击构建记录能正确跳转到构建详情页，页面参数正确。

### 步骤 2：后端增加构建详情扩展接口

当前 `GET /api/v1/builds/:id` 只返回 Build 模型的基本字段，前端需要关联信息。

- 新增 `BuildDetailResponse` 结构体，在 Build 基础上附加项目名称、环境名称、触发者用户名。
- 修改 `build_handler.go` 中的 `GetByID` 方法，查询并组装这些关联信息返回。
- 新增 `GET /api/v1/builds/:id/log` 接口，返回完整日志文件文本内容（用于非 WebSocket 场景下一次性加载历史日志）。

**完成标准**：`GET /builds/:id` 返回含 `project_name`、`environment_name`、`triggered_by_name` 的完整响应。日志接口可正常返回日志文本。

### 步骤 3：重写构建详情页前端

将 `builds/detail.tsx` 从占位页面重写为完整的构建详情页：

- **顶部区域**：构建标题 `Build #N`、返回按钮、操作按钮组（取消/重新构建/部署/回滚/下载产物）。
- **构建信息卡片**：显示项目名称、环境名称、触发方式、提交哈希、提交信息、触发者、创建时间、开始时间、结束时间、耗时。
- **构建状态时间线**：以可视化方式展示构建进度（pending → cloning → building → deploying → success/failed），当前阶段高亮、已完成阶段打钩、失败阶段标红。
- **构建日志区域**：使用增强版 `BuildLogViewer` 组件。
- 页面需根据构建状态显示不同操作按钮：
  - `running/cloning/building/deploying` 状态：显示"取消"按钮。
  - `success` 状态：显示"下载产物"、"部署"、"回滚"按钮。
  - `failed` 状态：显示"重新构建"按钮。
- 当构建正在运行时，通过 WebSocket 实时更新状态和日志。构建完成后自动停止 WebSocket 并刷新数据。

**完成标准**：构建详情页展示完整信息、操作按钮根据状态可用、日志实时滚动、产物可下载。

### 步骤 4：增强 BuildLogViewer 组件

在现有组件基础上增加以下功能：

- **ANSI 颜色解析**：安装 `ansi-to-html` 库或自实现简易解析器，将 ANSI 转义序列渲染为对应颜色的 HTML（支持常见前景色、加粗、ESC ERROR 高亮）。
- **日志搜索**：顶部增加搜索输入框，匹配行高亮显示，支持上/下导航跳转匹配项。
- **自动滚动控制**：增加"跟随最新"切换按钮。用户手动滚动时停止自动滚动，点击按钮恢复。
- **全屏模式**：增加全屏按钮，日志区域可展开到全屏模式。
- 优化性能：日志行数过多时（>5000 行）使用虚拟滚动或分段渲染。

**完成标准**：ANSI 颜色正确渲染，搜索可定位跳转，自动滚动可控，大量日志不卡顿。

### 步骤 5：后端增加重新构建接口

当前缺少"重新构建"功能，需要：

- 新增 `POST /api/v1/builds/:id/retry` 接口，基于指定构建的同项目、同环境创建新构建并提交调度。

**完成标准**：前端可通过"重新构建"按钮对 failed 构建重试，创建新的 build 记录并开始执行。

### 步骤 6：集成验证

- 前端 lint 通过 (`cd web && bun run lint`)。
- 后端编译通过 (`go build ./...`)。
- 手动验证：从项目详情 → 触发构建 → 跳转构建详情 → 查看实时日志 → 构建完成 → 下载产物（如有）→ 部署操作。
