# agent-context — 协议

## 何时使用

已安装 `ac-workflow` Skill 后，在对话中触发协作动作。这些**不是** CLI 子命令。

## 协议一览

| Protocol | 时机 | 前置 |
| --- | --- | --- |
| `init` | 建立协作规则 / SCOPE | 无 |
| `plan` | 新需求正式拆分 | 无冲突的当前计划 |
| `replan` | 未执行计划重做 | 目标为 `未执行` |
| `implement` | 按当前计划实施 | 当前计划存在且 `未执行` |
| `patch` | 已执行计划上增量修复 | 当前为 `已执行` |
| `rush` | 清晰小任务直接规划并实施 | 无未实施计划 |
| `review` | 第三方审查当前计划/实现 | 计划存在且校验通过 |
| `done` | 确认完成后归档 | 当前为 `已执行`（随后调 CLI `done`） |

Skill 的 `init` 与 CLI `agent-context init` 不同：前者偏协作约定，后者初始化 SCOPE。

## 核心规则

- 每个 SCOPE 同时最多一个当前计划
- 状态仅 `未执行` | `已执行`
- 影响范围不含 `.agent-context/` 下文件
- 有活跃且已执行计划时，再提变更优先 `patch`；`implement` 不接受追加需求

## 说法示例

```text
为“增加导出 Excel 功能”创建计划
按当前计划开始实现
给当前计划补一个 patch
当前计划已经真正完成，归档它
```
